环境配置

  • 用的是ubuntu 22.04, gcc和g++的版本都太高了，我参考网上的文章装了个gcc-7，然后通过下面这个设置提高优先级，以后出问题了记得改回去
    update-alternatives –install /usr/bin/gcc gcc /usr/bin/gcc-7 50
  • make版本也要降低，装了个4.2.1的
  • make还有bug: https://github.com/openjdk/panama-foreign/commit/af5c725b
  • bash ./configure –with-debug-level=slowdebug –with-jvm-variants=server –with-native-debug-symbols=internal再make images
  • 用vscode配合gdb调试，但是也有坑,gdb会捕获到SIGSEGV然后提示Segmentation fault，解决方法为在jdk根目录下新建一个.gdbinit文件，写入以下内容，然后在launch.json里面执行这个文件

HotSpot

对象头

  • markOop.hpp里面的markOopDesc类，”OOP” means “Ordinary Object Pointers”

log

  • JDK9之后可以在vmoptions里加上-Xlog:gc查看gc的日志

监控软件

  • jps: 最基础的查vmid的
  • jstat: 比如用jstat -gcutil vmid获取当前各个内存区域的使用情况以及gc次数和耗时
  • jinfo: 实时查看和调整虚拟机各项参数
  • jmap: 用于生成堆转储快照
  • jhat: (Java Heap Analysis Tool)与jmap搭配使用，来分析jmap生成的堆转储快照，但是一般不会用这玩意
  • jstack: 用于生成虚拟机当前时刻的线程快照(一般称为threaddump或者javacore文件)。
